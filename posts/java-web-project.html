Project Introduction

FishStore is an e-commerce platform specializing in high-quality live fresh fish, built with modern Java Web technology. The project offers a convenient, fast, and reliable online shopping experience for food lovers of fresh fish.

Strategic Goals

FishStore was created with the mission of connecting consumers with high-quality live fresh fish sources, while supporting fishermen and suppliers in product distribution.

Super-fast Delivery

  • Delivery within 2 hours in the city center
  • 100% freshness guarantee
  • Professional packaging

Smart AI Consulting

  • 24/7 virtual assistant
  • Consultation based on real-time inventory
  • Recipe suggestions

Quality Assurance

  • Strict inspection
  • Clear origin tracking
  • Food safety and hygiene

System Architecture

MVC Model

The system is built on the standard MVC (Model-View-Controller) model, ensuring clear separation between components and easy maintenance and expansion.

View Layer

  • JSP + JSTL
  • HTML5, CSS3, JavaScript

  • Bootstrap Components

Controller Layer

  • Java Servlets
  • Filter & Authentication
  • Request Routing
  • Session Management

Model Layer

  • DAO Pattern
  • SQL Server Database
  • POJO Classes
  • Connection Pooling

Key Components

Java Servlet JSP/JSTL SQL Server JDBC HTML5/CSS3 JavaScript Google Gemini AI RESTful API

System Features

Shopping System

Provides all features of a modern e-commerce system:

  • Diverse product catalog: Freshwater fish, sea fish, premium seafood
  • Smart search: By name, category, price range
  • Flexible shopping cart: Add/remove/edit product quantity
  • Diverse payment: COD, bank transfer
  • Order tracking: Real-time status

Smart AI Consulting

Integrates Google Gemini AI to provide a smart consulting experience:

  • 24/7 Chatbot: Customer support anytime
  • Product consulting: Based on real-time inventory
  • Recipe suggestions: How to cook delicious dishes from fish
  • Technical support: Proper fish preservation and preparation

Multi-role Management

The system offers detailed authorization for different roles:

  • Customer: Shopping, order tracking, reviews
  • Staff: Order management, customer support
  • Administrator: Manage the entire system, products, users

Database Design

Relational Model

The system uses SQL Server with standardized relational tables:

Customer

  • KHACH_HANG (Customer)
  • Role classification
  • Login authentication

Product

  • DANH_MUC (Category)
  • LOAI_CA (Fish Type)
  • Inventory management

Order

  • DON_HANG (Order)
  • CHI_TIET_DON_HANG (Order Detail)
  • GIO_HANG (Shopping Cart)

Data Security

  • Password encryption: Using BCrypt
  • SQL Injection Prevention: Prepared Statement
  • XSS Protection: Input validation & output encoding
  • Session Management: Secure session handling

Challenges & Solutions

Issues Encountered

1. Integrating AI with Real Data

Problem: How can AI consult based on real-time inventory instead of virtual information?

Solution: Building a GeminiService that connects directly to the database, providing real product context to the AI.

2. Session & Authentication Management

Problem: How to secure user information and manage work sessions?

Solution: Implementing AuthFilter, using a combination of Session and Cookie, and detailed authorization.

3. Performance & Scalability

Problem: Ensuring performance when the number of users increases?

Solution: Using connection pooling, caching, and database query optimization.
